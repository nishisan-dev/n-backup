# v3.0.0 — Slots, Per-N-Chunk Rotation & WebUI

Evolução breaking do protocolo para tornar slots first-class, adicionar rastreabilidade chunk→slot, per-N-chunk source-port rotation e observabilidade rica de slots na WebUI.

> [!IMPORTANT]
> **API Breaking Change**: `ProtocolVersion` sobe de `0x04` para `0x05`. Agents e servers devem ser atualizados simultaneamente. Sem fallbacks.

## User Review Required

> [!CAUTION]
> A refatoração da `ParallelSession` (8+ `sync.Map` → `[]Slot`) afeta o core do handler. Apesar de ser uma simplificação, é o componente mais crítico e com maior blast radius. Sugestão: executar esta fase isolada com testes extensivos antes de avançar.

> [!IMPORTANT]
> O `ChunkHeader` cresce de 8B para 9B com o `SlotID`. Isso é um wire-format breaking change que afeta **todo** o pipeline: agent dispatcher, server `receiveParallelStream`, `ChunkBuffer`, `GapTracker`, `ChunkAssembler`.

### Fase 2: Server — Struct `Slot` e Pré-alocação

Refatoração do core do handler para slots explícitos.

---

#### [MODIFY] [handler.go](file:///home/lucas/Projects/n-backup/internal/server/handler.go)

**2a. Criar `type Slot struct`** — unifica 8+ `sync.Map` em struct coesa:
```go
// SlotStatus representa o estado observável de um slot.
type SlotStatus string

const (
    SlotIdle         SlotStatus = "idle"
    SlotReceiving    SlotStatus = "receiving"
    SlotDisconnected SlotStatus = "disconnected"
    SlotDisabled     SlotStatus = "disabled"
)

// Slot representa um slot de stream paralelo com estado unificado.
type Slot struct {
    mu            sync.Mutex
    Index         uint8
    Conn          net.Conn
    Cancel        context.CancelFunc
    Status        SlotStatus
    Offset        int64           // bytes recebidos acumulados
    TrafficIn     atomic.Int64    // bytes no intervalo (para stats/flow rotation)
    TickBytes     int64           // bytes observados no último tick
    LastAct       atomic.Int64    // UnixNano último I/O
    ConnectedAt   time.Time
    Reconnects    atomic.Int32
    SlowSince     time.Time       // início de degradação contínua
    LastReset     time.Time       // último flow rotation
    // Chunk stats (novidade)
    ChunksReceived    atomic.Uint32
    ChunksLost        atomic.Uint32
    ChunksRetransmit  atomic.Uint32
    LastChunkSeq      atomic.Uint32
}
```

**2b. Refatorar `ParallelSession`** — substituir 8+ `sync.Map` por `[]Slot`:
```go
type ParallelSession struct {
    // ... campos existentes que não mudam ...
    Slots         []*Slot     // pré-alocado com MaxStreams slots
    // REMOVE: StreamConns, StreamOffsets, StreamTrafficIn, StreamLastAct,
    //         StreamCancels, StreamSlowSince, StreamLastReset,
    //         StreamConnectedAt, StreamReconnects, StreamTickBytes,
    //         RotatePending
}
```

**2c. Adaptar `handleParallelBackup`** — pré-alocar slots:
```go
// No handleParallelBackup, após criar ParallelSession:
slots := make([]*Slot, pi.MaxStreams)
for i := uint8(0); i < pi.MaxStreams; i++ {
    slots[i] = &Slot{
        Index:  i,
        Status: SlotIdle,
    }
}
pSession.Slots = slots
```

**2d. Adaptar `handleParallelJoin`** — operar sobre `Slot`:
- Substituir todas as referências a `StreamConns.Load/Store`, `StreamOffsets.LoadOrStore`, etc. por acesso direto a `pSession.Slots[pj.StreamIndex]`
- Transições de estado: `idle` → `receiving` no join, `receiving` → `disconnected` no disconnect

**2e. Adaptar `evaluateFlowRotation`** — iterar sobre `[]Slot`:
- Substituir `ps.StreamTrafficIn.Range(...)` por `for _, slot := range ps.Slots { ... }`
- Acessar `slot.TrafficIn.Swap(0)` diretamente

**2f. Adaptar `receiveParallelStream`** — chunk stats por slot:
- No loop de recepção de chunks, após sucesso: `slot.ChunksReceived.Add(1)`, `slot.LastChunkSeq.Store(hdr.GlobalSeq)`
- O `SlotID` vindo no `ChunkHeader` permite rastrear origem em retransmissões

**2g. Processar `ControlSlotPark`/`ControlSlotResume`** no control channel:
- No handler do control channel, ao receber `ControlSlotPark{SlotID}`: atualizar `slot.Status = SlotDisabled`
- Ao receber `ControlSlotResume{SlotID}`: atualizar `slot.Status = SlotIdle`
